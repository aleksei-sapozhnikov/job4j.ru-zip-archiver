Консольный zip-архиватор

Задание:

  1. Задана директория проекта, например: c:\project\
  2. В качестве ключей передаются расширения файлов, которые должны попасть в архив.
  3. Архив должен сохранять структуру проекта: вложенные папки и файлы в них.
  4. Для архивации использовать класс java.util.zip.ZipOutputStream.
  5. Запуск архивации: java-jar zip-archiver.jar -a archive -s /home/john/project -o /home/john/my_project.zip -e xml,pdf,txt
    • zip-archiver.jar: скомпилированный файл приложения.
    • -a archive: действие - архивировать (action).
    • -s /home/john/project: источник, папка с проектом (source).
    • -o /home/john/my_project.zip: результирующий файл (output).
    • -e xml,pdf,txt: Расширения файлов, которые нужно включить в проект (extensions).
Результат:
  1. Проект создан с использованием сборщика Maven, который запаковывает проект в jar-файл.
  2. Отдельный класс анализирует и проверяет переданные параметры приложения.
  3. При запуске без параметров приложение выводит справку по использованию.
  4. Написаны автоматические тесты с использованием JUnit, Mockito. Один из тестов вначале архивирует проект, затем разархивирует и проверяет - получилось ли в конце то же самое, что было изначально.
  5. Структура проекта для тестов создается во временной директории во время самих тестов. Нет нужды отправлять на Github заранее заготовленные файлы.
